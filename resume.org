mkdir tuto

</pre></div>

<div class="highlight highlight-sh"><pre>
git clone &lt;urldudepot&gt;
</pre></div>
<h1>
<a name="user-content-2-utilisation-basique-de-git" class="anchor" href="#2-utilisation-basique-de-git"><span class="octicon octicon-link"></span></a>2 Utilisation basique de git</h1>
<h2>
<a name="user-content-21-ajout-dun-fichier" class="anchor" href="#21-ajout-dun-fichier"><span class="octicon octicon-link"></span></a>2.1 Ajout d’un fichier</h2>
<div class="highlight highlight-sh"><pre>
git add resume.org
</pre></div>
<h2>
<a name="user-content-22-afficher-le-statut-actuel-du-d%C3%A9p%C3%B4t" class="anchor" href="#22-afficher-le-statut-actuel-du-d%C3%A9p%C3%B4t"><span class="octicon octicon-link"></span></a>2.2 Afficher le statut actuel du dépôt</h2>
<p>Où l’on voit que le fichier a bien été ajouté pour faire partie du
  prochain <i>commit</i>.</p>
<div class="highlight highlight-sh"><pre>
git status
</pre></div>
<h2>
<a name="user-content-23-r%C3%A9aliser-un-commit-pour-sauvegarder-une-%C3%A9tape" class="anchor" href="#23-r%C3%A9aliser-un-commit-pour-sauvegarder-une-%C3%A9tape"><span class="octicon octicon-link"></span></a>2.3 Réaliser un commit pour sauvegarder une étape</h2>
<p>Pour avoir un peu d’aide.</p>
<div class="highlight highlight-sh"><pre>
git <span class="nb">help </span>commit
</pre></div>
<p>La commande commit permet de valider nos modifications et de leur
  associer un message explicatif.</p>
<div class="highlight highlight-sh"><pre>
git commit -m <span class="s2">"Ajout d'un compte-rendu"</span>
</pre></div>
<p>Git nous informe alors du résultat de notre commande (un fichier modifié).</p>
<pre>
[master (root-commit) 2bb1fab] Ajout d'un compte-rendu
1 files changed, 13 insertions(+), 0 deletions(-)
create mode 100644 resume.org
</pre>
<h2>
<a name="user-content-24-v%C3%A9rifier-les-derni%C3%A8res-modifications" class="anchor" href="#24-v%C3%A9rifier-les-derni%C3%A8res-modifications"><span class="octicon octicon-link"></span></a>2.4 Vérifier les dernières modifications</h2>
<div class="highlight highlight-sh"><pre>
git log
</pre></div>
<p>Commit de toutes les modifications sur les fichiers précédemment ajoutés.</p>
<div class="highlight highlight-sh"><pre>
git commit -a -m <span class="s2">"dernières modifs"</span>
</pre></div>
<h1>
<a name="user-content-3-commencer-un-projet-caml" class="anchor" href="#3-commencer-un-projet-caml"><span class="octicon octicon-link"></span></a>3 Commencer un projet caml</h1>
<p>Création d’un fichier astro.ml</p>
<div class="highlight highlight-ocaml"><pre>
<span class="c">(* Graphics.close_graph ();; *)</span>
<span class="nn">Graphics</span><span class="p">.</span><span class="n">open_graph</span> <span class="s2">" 350x350"</span><span class="o">;;</span>

<span class="n">print_string</span> <span class="s2">"          *** космонавт ***</span><span class="se">\n</span><span class="s2">"</span><span class="o">;;</span>
<span class="n">print_string</span> <span class="s2">"       Salut jeune astronaute</span><span class="se">\n</span><span class="s2">"</span><span class="o">;;</span>
</pre></div>
<p>À compiler comme toplevel comme ceci:</p>
<div class="highlight highlight-sh"><pre>
ocamlmktop graphics.cma astro.ml -o FlappyCandyAstroCrush
</pre></div>
<p>Rajoutons un de nos fichiers d’images :</p>
<div class="highlight highlight-sh"><pre>
git add images50.ml
git commit -m <span class="s2">"ajout fichier images"</span>
</pre></div>
<p>Et apprenons à afficher une image, dans un fichier dessiner.ml.</p>
<h2>
<a name="user-content-31-le-fichier-dessinerml" class="anchor" href="#31-le-fichier-dessinerml"><span class="octicon octicon-link"></span></a>3.1 Le fichier dessiner.ml</h2>
<p>Nous manipulons les images sous forme de matrices (des tableaux de
  tableaux) dont les éléments ont soit des couleurs soit des paires
  couleur, opacité.</p>
<p>On commence par une fonction pour récupérer les dimensions d’une
  matrice (hauteur et largeur).</p>
<div class="highlight highlight-ocaml"><pre>
<span class="k">let</span> <span class="n">dimensions</span> <span class="n">m</span> <span class="o">=</span>
  <span class="k">let</span> <span class="n">h</span> <span class="o">=</span> <span class="nn">Array</span><span class="p">.</span><span class="n">length</span> <span class="n">m</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">l</span> <span class="o">=</span> <span class="nn">Array</span><span class="p">.</span><span class="n">length</span> <span class="n">m</span><span class="o">.(</span><span class="mi">0</span><span class="o">)</span> <span class="k">in</span>
  <span class="o">(</span><span class="n">l</span><span class="o">,</span> <span class="n">h</span><span class="o">);;</span>
</pre></div>
<p>Le module Graphics permet de composer de nouvelles couleurs à partir
  de leurs composantes rouge verte et bleue (fonction Graphics.rgb).
  L’opération inverse n’est par contre pas fournie. Programmons la.</p>
<div class="highlight highlight-ocaml"><pre>
<span class="k">let</span> <span class="n">decompose</span> <span class="n">c</span> <span class="o">=</span>
  <span class="k">let</span> <span class="n">b</span> <span class="o">=</span> <span class="n">c</span> <span class="ow">mod</span> <span class="mi">256</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">v</span> <span class="o">=</span> <span class="o">(</span><span class="n">c</span> <span class="o">/</span> <span class="mi">256</span><span class="o">)</span> <span class="ow">mod</span> <span class="mi">256</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">r</span> <span class="o">=</span> <span class="o">(</span><span class="n">c</span> <span class="o">/</span> <span class="mi">65536</span><span class="o">)</span> <span class="ow">mod</span> <span class="mi">256</span> <span class="k">in</span>
  <span class="o">(</span><span class="n">r</span><span class="o">,</span><span class="n">v</span><span class="o">,</span><span class="n">b</span><span class="o">);;</span>
</pre></div>
<p>On s’en sert pour mélanger deux couleurs selon un certain niveau
  d’opacité allant de 255 (opaque) à 0 (transparent). Il y a un bug dans
  le code suivant, saurez-vous le trouver ? (Il est corrigé dans le dépôt).</p>
<div class="highlight highlight-ocaml"><pre>
<span class="k">let</span> <span class="n">melanger</span> <span class="n">c1</span> <span class="n">c2</span> <span class="n">op</span> <span class="o">=</span>
  <span class="k">let</span> <span class="o">(</span><span class="n">r1</span><span class="o">,</span><span class="n">v1</span><span class="o">,</span><span class="n">b1</span><span class="o">)</span> <span class="o">=</span> <span class="n">decompose</span> <span class="n">c1</span> <span class="k">in</span>
  <span class="k">let</span> <span class="o">(</span><span class="n">r2</span><span class="o">,</span><span class="n">v2</span><span class="o">,</span><span class="n">b2</span><span class="o">)</span> <span class="o">=</span> <span class="n">decompose</span> <span class="n">c2</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">mix</span> <span class="n">a</span> <span class="n">b</span> <span class="n">op</span> <span class="o">=</span>
    <span class="o">(</span><span class="n">op</span> <span class="o">*</span> <span class="n">a</span> <span class="o">+</span> <span class="o">(</span><span class="mi">255</span> <span class="o">-</span> <span class="n">op</span> <span class="o">*</span> <span class="n">b</span><span class="o">))/</span><span class="mi">255</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">r3</span> <span class="o">=</span> <span class="n">mix</span> <span class="n">r1</span> <span class="n">r2</span> <span class="n">op</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">v3</span> <span class="o">=</span> <span class="n">mix</span> <span class="n">v1</span> <span class="n">v2</span> <span class="n">op</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">b3</span> <span class="o">=</span> <span class="n">mix</span> <span class="n">b1</span> <span class="n">b2</span> <span class="n">op</span> <span class="k">in</span>
  <span class="nn">Graphics</span><span class="p">.</span><span class="n">rgb</span> <span class="n">r3</span> <span class="n">v3</span> <span class="n">b3</span><span class="o">;;</span>
</pre></div>
<p>La fonction matrixmap2 que nous avons construit en TP
  s’obtient très simplement comme ceci.</p>
<div class="highlight highlight-ocaml"><pre>
<span class="k">let</span> <span class="n">arraymap2</span> <span class="n">f</span> <span class="n">ta</span> <span class="n">tb</span> <span class="o">=</span>
  <span class="k">let</span> <span class="n">g</span> <span class="n">i</span> <span class="n">b</span> <span class="o">=</span> <span class="n">f</span> <span class="n">ta</span><span class="o">.(</span><span class="n">i</span><span class="o">)</span> <span class="n">b</span> <span class="k">in</span>
  <span class="nn">Array</span><span class="p">.</span><span class="n">mapi</span> <span class="n">g</span> <span class="n">tb</span><span class="o">;;</span>

<span class="k">let</span> <span class="n">matrixmap2</span> <span class="n">f</span> <span class="o">=</span>
  <span class="n">arraymap2</span> <span class="o">(</span><span class="n">arraymap2</span> <span class="n">f</span><span class="o">);;</span>
</pre></div>
<p>Forts de la fonction melanger et de matrixmap2, il est facile d’écrire le code permettant de
  dessiner une image en tenant compte de la transparence.</p>
<div class="highlight highlight-ocaml"><pre>
<span class="k">let</span> <span class="n">dessiner_image</span> <span class="n">img</span> <span class="n">x</span> <span class="n">y</span> <span class="o">=</span>
  <span class="k">let</span> <span class="o">(</span><span class="n">largeur</span><span class="o">,</span><span class="n">hauteur</span><span class="o">)</span> <span class="o">=</span> <span class="n">dimensions</span> <span class="n">img</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">fond</span> <span class="o">=</span> <span class="nn">Graphics</span><span class="p">.</span><span class="n">dump_image</span> <span class="o">(</span><span class="nn">Graphics</span><span class="p">.</span><span class="n">get_image</span> <span class="n">x</span> <span class="n">y</span> <span class="n">largeur</span> <span class="n">hauteur</span><span class="o">)</span> <span class="k">in</span>
  <span class="k">let</span> <span class="n">melanger_tordu</span> <span class="o">(</span><span class="n">c1</span><span class="o">,</span><span class="n">o</span><span class="o">)</span> <span class="n">c2</span> <span class="o">=</span> <span class="n">melanger</span> <span class="n">c1</span> <span class="n">c2</span> <span class="n">o</span> <span class="k">in</span>
  <span class="nn">Graphics</span><span class="p">.</span><span class="n">draw_image</span> <span class="o">(</span><span class="nn">Graphics</span><span class="p">.</span><span class="n">make_image</span> <span class="o">(</span><span class="n">matrixmap2</span> <span class="n">melanger_tordu</span> <span class="n">img</span> <span class="n">fond</span><span class="o">))</span> <span class="n">x</span> <span class="n">y</span><span class="o">;;</span>
</pre></div>
<p>Ceci termine le fichier dessiner.ml. On l’ajoute au dépôt.</p>
<div class="highlight highlight-sh"><pre>
git add dessiner.ml
git commit -a -m <span class="s2">"ajout de la fonction de dessin d'images"</span>
</pre></div>
<p>Le premier essai révèle un bug (notamment visible au fait que la
  transparence totale ne survient pas). On le corrige facilement.</p>
<div class="highlight highlight-sh"><pre>
git commit dessiner.ml -m <span class="s2">"correction d'un bug dans le calcul de la transparence"</span>
</pre></div>
<h2>
<a name="user-content-32-le-fichier-directionsml" class="anchor" href="#32-le-fichier-directionsml"><span class="octicon octicon-link"></span></a>3.2 Le fichier directions.ml</h2>
<p>Comme les directions (les quatre points
  cardinaux) utilisées dans les déplacements sont nécessaires
  à la fois pour l’interprète et pour les graphismes, on les place dans
  un module à part.</p>
<div class="highlight highlight-ocaml"><pre>
<span class="c">(* direction.ml *)</span>
<span class="k">type</span> <span class="n">direction</span> <span class="o">=</span> <span class="nc">N</span> <span class="o">|</span> <span class="nc">S</span> <span class="o">|</span> <span class="nc">E</span> <span class="o">|</span> <span class="nc">O</span><span class="o">;;</span>
</pre></div>
<h2>
<a name="user-content-33-un-premier-interpr%C3%A8te-pour-nos-tests-le-fichier-interpreteml" class="anchor" href="#33-un-premier-interpr%C3%A8te-pour-nos-tests-le-fichier-interpreteml"><span class="octicon octicon-link"></span></a>3.3 Un premier interprète pour nos tests (le fichier interprete.ml)</h2>
<p>On rajoute un petit interpète maison (à augmenter à partir ce que vous
  avez appris à faire en TD/TP). Les instructions sont minimales : se
  déplacer dans les quatres directions, regrouper les instructions en blocs.</p>
<h2>
<a name="user-content-34-retour-au-fichier-principal-astroml" class="anchor" href="#34-retour-au-fichier-principal-astroml"><span class="octicon octicon-link"></span></a>3.4 Retour au fichier principal (astro.ml)</h2>
<p>Puis on commence
  à améliorer le jeu, voir directement dans les logs git, en particulier
  tout le fichier astro.ml.</p>
<h2>
<a name="user-content-35-compilation-compl%C3%A8te" class="anchor" href="#35-compilation-compl%C3%A8te"><span class="octicon octicon-link"></span></a>3.5 Compilation complète</h2>
<p>La compilation se fait alors comme ceci:</p>
<div class="highlight highlight-sh"><pre>
ocamlmktop directions.cmo interprete.cmo graphics.cma images50.cmo dessiner.ml <span class="se">\</span>
astro.ml -o FlappyCandyAstroCrush
</pre></div>
<p>Il est nécessaire de donner les modules dans l’ordre de
  leur utilisation (il faut respecter l’ordre de dépendance entre
  modules). On peut donner à cette ligne de commande les modules
  compilés, ou bien les fichiers sources l’un et l’autre fonctionne pour
  chaque fichier.</p>
<p>À l’ouverture de notre REPL (ou top-level) OCaml personnalisé, il
  reste malheureusement à ouvrir deux modules pour peupler l’espace de
  nom racine avec les types de l’interprète (et tout particulièrement la syntaxe du
  mini-langage) et pour donner accès à la
  fonction de jeu. Après seulement on peut commencer à jouer.</p>
<div class="highlight highlight-ocaml"><pre>
<span class="k">open</span> <span class="nc">Interprete</span><span class="o">;;</span>
<span class="k">open</span> <span class="nc">Astro</span><span class="o">;;</span>
<span class="n">jouer</span> <span class="o">(</span><span class="nc">Bloc</span> <span class="o">[</span><span class="nc">Est</span><span class="o">;</span> <span class="nc">Est</span><span class="o">;</span> <span class="nc">Nord</span><span class="o">]);;</span>
<span class="n">jouer</span> <span class="o">(</span><span class="nc">Bloc</span> <span class="o">[</span><span class="nc">Ouest</span><span class="o">;</span> <span class="nc">Nord</span><span class="o">;]);;</span>
</pre></div></article>
